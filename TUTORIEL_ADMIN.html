Tutoriel Espace Admin

Carte Limap — Guide de gestion des donnees, des demandes clients et des utilisateurs.

1. Acces a l'espace admin

Ouvrez la page suivante dans votre navigateur :

https://limap.fr/Carte/admin/index.html

  1. Saisissez votre identifiant et votre mot de passe admin.
  2. Cliquez sur Se connecter.
    Vous arrivez sur le tableau de bord avec tous les onglets de gestion.
Si vous etes deja connecte sur une autre session (client par exemple), commencez par vous deconnecter de cette session avant d'acceder a l'admin.
Nouveau : depuis la carte publique (https://limap.fr/Carte/), un client peut se connecter via le menu gauche puis ouvrir son espace avec le bouton « Acceder a mon espace client ».

1 bis. Carte publique vs mode prive client

ModeQui y accedeDonnees visibles
Public Visiteur non connecte Uniquement les clients/fournisseurs conformes aux regles de consentement.
Prive client Client connecte depuis la carte Tous les clients/fournisseurs (y compris non valides consentement) pour travail interne.
Admin Compte admin Redirection vers l'espace admin complet.
Important : la vue publique reste filtree par consentement. Le mode prive est reserve aux utilisateurs authentifies.

3. Gestion des clients

Un client est une structure adherente (AMAP, groupement d'achat, etc.) qui a acces a son propre espace.

Creer un nouveau client

  1. Onglet Clients → remplissez le formulaire.
    Nom, type, adresse, email, site web, logo, description.
  2. Cliquez sur Enregistrer le client.
  3. Creez ensuite au moins un compte utilisateur pour ce client.
    Onglet Utilisateurs → Nouvel utilisateur client (cf. section 7).

Lier un fournisseur a un client

Une fois le client cree, son perimetre fournisseurs se construit de deux facons :

  • Manuellement en admin : onglet Fournisseurs, ouvrez une fiche fournisseur et cochez le client correspondant dans la liste des liens.
  • Via une demande de rattachement client : le client en fait la demande depuis son espace, vous l'approuvez (cf. section 9).
Le logo du client doit etre uploade via le champ "Logo" du formulaire. Il sera automatiquement normalise et integre dans la carte publique.

4. Gestion des referentiels (Categories, Labels, Types)

Ces trois referentiels definissent les valeurs disponibles lors de la saisie d'un fournisseur.

ReferentielRoleExemple
CategoriesActivites du fournisseurMaraichage, Apiculture, Elevage bovin
LabelsCertifications et engagementsBio AB, Nature et Progres, Commerce equitable
TypesNature juridique du fournisseurGAEC, EARL, SCEA, Artisan

Pour ajouter ou modifier une valeur : ouvrez l'onglet correspondant, remplissez le formulaire et enregistrez. Les modifications sont immediatement disponibles dans tous les formulaires.

Evitez de supprimer un referentiel utilise par des fournisseurs existants : cela peut retirer des donnees des fiches producteurs.

5. Gestion des fournisseurs

Un fournisseur est une fiche producteur unique dans la base, potentiellement partagee entre plusieurs clients.

Creer un fournisseur

  1. Onglet Fournisseurs → formulaire en bas "Nouveau fournisseur".
  2. Renseignez au minimum le nom, la ville et le type.
    Completez avec adresse, coordonnees GPS, contact, activites, labels, site web, logo, description.
  3. Selectionnez le(s) client(s) auxquels rattacher ce fournisseur.
  4. Enregistrez. La fiche est immediatement visible pour les clients selectionnes.

Modifier un fournisseur

Cliquez sur le fournisseur dans la liste → le formulaire se pre-remplit → modifiez → Enregistrer. Les modifications sont repercutees sur la carte publique si le fournisseur est marque "public".

Doublons et fiche globale

Avant de creer un nouveau fournisseur, verifiez qu'il n'existe pas deja (recherche par nom, ville, email). Une fiche dupliquee degrade la qualite des donnees de toute la plateforme.

6. Import Excel massif

L'onglet Import permet d'importer une grande quantite de fournisseurs depuis un fichier Excel, avec detection automatique des conflits.

  1. Selectionnez le client cible dans la liste deroulante.
  2. Choisissez le fichier Excel (.xlsx) prepare selon le modele attendu.
    Colonnes typiques : Nom, Adresse, Ville, Code postal, Tel, Email, Site web, Activites, Type, Labels.
  3. Cliquez sur Analyser.
    L'outil detecte les conflits potentiels (fournisseur deja existant portant un nom similaire).
  4. Verifiez les lignes en conflit et choisissez pour chacune : ignorer, fusionner ou creer quand meme.
  5. Lancez l'import.
Nouveau client avec une base existante ?
Si un client vous envoie un fichier Excel avec sa liste de producteurs, c'est de loin la methode la plus efficace. Un seul import cree tous les fournisseurs et les rattache automatiquement au bon client. Conseillez toujours cette option plutot que la saisie manuelle ou les demandes de creation une par une.

7. Gestion des utilisateurs

Roles disponibles

RolePerimetre
adminAcces complet a l'espace admin. A reserver aux administrateurs de la plateforme.
client_managerAcces client : peut tout faire dans son espace (modifier fiches, envoyer demandes).
client_editorAcces client limite : peut modifier les fiches mais pas supprimer ni gerer les utilisateurs.
client_readerLecture seule dans l'espace client.

Creer un utilisateur client

  1. Onglet Utilisateurs → section "Nouvel utilisateur client".
  2. Selectionnez le client auquel rattacher ce compte.
  3. Saisissez l'identifiant, l'email et le mot de passe provisoire.
  4. Choisissez le role (client_manager en general pour le contact principal).
  5. Enregistrez. Communiquez l'URL et les identifiants au client.

Reinitialiser un mot de passe

Deux options :

  • Par l'admin : onglet Utilisateurs → trouvez l'utilisateur → cliquez "Modifier le mot de passe".
  • Par le client lui-meme : depuis la page de connexion → "Mot de passe oublie" → un lien est envoye par email.

8. Traiter les demandes de modification (champs globaux)

Quand un client propose une correction sur la fiche globale d'un fournisseur (adresse, tel, email, activites...), une demande apparait dans l'onglet Demandes.

  1. Onglet Demandes → section "Demandes de modification (champs globaux)".
  2. Filtrez par statut ou par client si necessaire.
  3. Pour chaque demande, comparez l'ancienne et la nouvelle valeur.
  4. Approuver : la valeur est immediatement mise a jour sur la fiche fournisseur.
  5. Refuser : la valeur reste inchangee. Vous pouvez ajouter une note de refus.
Vous pouvez approuver ou refuser plusieurs demandes d'un coup avec les boutons "Approuver la selection" et "Refuser la selection".

9. Traiter les demandes de creation de fournisseurs

Un client peut proposer un nouveau fournisseur a ajouter dans la base.

  1. Onglet Demandes → section "Demandes de creation de fournisseurs".
  2. Consultez les informations saisies par le client.
    Nom, adresse, contact, activites, notes.
  3. Verifiez qu'il n'existe pas deja dans la base (recherchez dans l'onglet Fournisseurs).
  4. Si la demande est valide → Approuver.
    Le fournisseur est cree automatiquement et lie au client demandeur. Vous pouvez ajuster ses informations ensuite dans l'onglet Fournisseurs.
  5. Si doublon ou information insuffisante → Refuser avec une note explicative.
    Le client verra le statut "Refusee" et votre note dans son historique.

10. Traiter les demandes de rattachement client-fournisseur

Un client peut demander a ce qu'un fournisseur deja existant dans la base soit rattache a sa structure.

  1. Onglet Demandes → section "Demandes de rattachement client-fournisseur".
  2. Verifiez le client, le fournisseur concerne et la note eventuelle du client.
  3. Approuver : le fournisseur est immediatement rattache au client. Il apparaitra dans son espace a la prochaine connexion.
  4. Refuser : le fournisseur n'est pas rattache. Ajoutez une note si necessaire.
C'est la bonne procedure pour eviter les doublons : le client identifie lui-meme le fournisseur existant et en demande le rattachement, plutot que de recrer une fiche.

11. Parametres

L'onglet Parametres regroupe les reglages globaux de la plateforme.

ParametreUtilite
URL publique des assets Base URL du serveur (ex : https://limap.fr/Carte). Utilise pour construire les URLs completes des images et logos dans la carte publique.
Emails de notification admin Adresses email qui recoivent les alertes lors de nouvelles demandes clients (modification, creation, rattachement). Separees par des virgules.
Icone par defaut (client / producteur) Icone affichee sur la carte quand un client ou un producteur n'a pas de logo personnalise.
Apres avoir modifie les parametres, rechargez la carte publique avec Ctrl+F5 pour verifier que les assets s'affichent correctement.

12. Export vers WordPress (procedure complete)

  1. Preparation avant export
    Dans Parametres, verifiez :
    - URL publique des assets = https://limap.fr/Carte
    - logos/images de couverture visibles sur quelques fiches test
    - les donnees a publier sont bien marquees publiques
  2. Exporter depuis Limap
    - Onglet Clients → Exporter clients CSV
    - Onglet Fournisseurs → Exporter fournisseurs CSV (pour un cycle complet)
    - Onglet Fournisseurs → Exporter fournisseurs modifies (pour une mise a jour incrementale)
  3. Aller dans WordPress
    Ouvrir WP All Import → Manage Imports.
  4. Choisir l'import preconfigure
    Il y a 2 imports deja configures :
    - un import clients
    - un import fournisseurs
    Selectionnez celui qui correspond au fichier exporte.
  5. Remplacer le fichier CSV
    Dans l'import choisi : cliquez sur Settings, chargez le nouveau fichier CSV, validez.
  6. Lancer l'import
    Cliquez sur Run Import et attendez la fin complete du traitement.
  7. Controle post-import
    Verifiez rapidement :
    - pages clients/producteurs,
    - slugs,
    - images,
    - activites/labels,
    - dernieres fiches modifiees.
Les CSV exportes par Limap sont en UTF-8 avec separateur point-virgule ;. Si WP All Import detecte mal le format, forcez UTF-8 et le separateur ;.

13. Bonnes pratiques admin

  • Toujours chercher avant de creer : verifiez qu'un fournisseur ou un client n'existe pas deja avant toute creation.
  • Traiter les demandes rapidement : les clients voient leur demande "En attente" et peuvent se decourager si aucun retour n'arrive.
  • Ajouter une note en cas de refus : une note claire aide le client a comprendre pourquoi sa demande a ete refusee et a soumettre une correction.
  • Ne pas partager le compte admin : chaque administrateur doit avoir son propre compte pour maintenir une traçabilite.
  • Sauvegarder regulierement : exportez ou sauvegardez la base de donnees avant toute operation massive (import Excel, suppressions en lot).
  • Verifier les emails de notification : tester la notification depuis Parametres → bouton "Envoyer un email de test".

14. En cas de probleme

  • Erreur apres une action : ouvrir la console du navigateur (F12 → Console) et noter le message d'erreur.
  • Page vide ou donnees manquantes : recharger avec Ctrl+F5. Si le probleme persiste, verifier les parametres et la connexion au serveur.
  • Import Excel bloque : verifier que le fichier est bien au format .xlsx et que les colonnes correspondent au modele attendu.
  • Email de notification non recu : tester depuis Parametres, puis verifier le dossier Spam et la configuration SMTP/mail du serveur OVH.
